# WBS 工作分解结构

// pm-scope/05 · 核心课时 · ~5min

cat concept.md

想象你要搬家——不会直接喊"搬！"，而是拆成：找搬家公司、打包厨房、打包卧室、搬运、验收。WBS 就是把项目大目标逐层拆解成可管理、可交付的小工作包的过程。

它是范围基准的核心组成部分，与 WBS 词典和范围说明书共同构成范围基准。

cat core.md

分解四原则

  • 100% 规则：子元素之和 = 父元素全部范围
  • 面向可交付成果：分解"东西"而非"动作"
  • 最底层是工作包：8-80 小时可估算
  • 每个元素只出现一次
grep -i "误解" pitfalls.md
WBS 是任务清单 → 错。分解的是可交付成果（名词），不是活动。"写代码"是活动，"登录模块"才是工作包。
WBS 要分解到最细 → 错。最底层是工作包（8-80h），再往下是活动，属于进度管理。
