## 五、架构决策记录（ADR）

> ADR-1～7 继承自参考站（已上线验证），ADR-8～10 为本站新增。

### ADR-1 · 不要后端，纯静态到底
自用/小范围学习，不做账号、不做 UGC。原生 HTML/CSS/JS，零依赖零构建，`file://` 双击可跑。

### ADR-2 · 托管用 GitHub Pages（public repo）
三端同一 URL，`git push` 即同步；纯静态照样 `file://` 离线跑。内容为行业通识，无隐私，public 可接受。**独立仓库，切勿与其它项目混仓**。

### ADR-3 · 内容即数据 + 模板渲染（引擎/内容分离）
每课一条结构化数据（五段模板），统一渲染器吐页面；内容用 `<script>` 注册进全局 registry（避开 `file://` fetch 的 CORS）。加内容 = 加数据不碰引擎；搜索索引自动生成。全局命名空间 **`PGF`**。

### ADR-4 · 进度同步 = GitHub 当后端（可选、自动无感）
浏览器存 fine-grained PAT，自动读写用户自己的 private repo 里的 `progress.json`；PAT 只授权该仓库 `Contents: Read/Write`、建议 90 天过期。**代码里永远不出现 token**。不配置则进度仅存本机 localStorage，完全不影响学习。

### ADR-5 · 冲突合并 = 按条目智能合并 + 软删除
写入先拉远端最新 → 按条目并集合并（同 id 以 `updatedAt` 新者为准）；偏好类单值"最后写入为准"；删除走墓碑（`removed: true` + 时间戳）；409 则重拉重并再写。

### ADR-6 · 进度数据模型（progress.json）
1. **术语收藏本**（核心，跨设备同步主要为它）；2. **课程已读标记 + 续读位置**（手动点"标记已读"，不做滚动自动判定）；3. **偏好**（主题/字号）。不做 SRS，数据结构预留。

### ADR-7 · 路线图优先、图书馆其次
首页主入口是线性学习路径「继续学习：你在第 X 步」；17 模块目录降级为查阅入口。未编写课程显示「待编写」并可跳过——结构先行，缺口可见。

### ADR-8 · 术语点查系统（本站灵魂之一，替代参考站 TTS 点读）
- 全站术语库 `content/terms.js`：每条 = id + 名称 + 大白话定义 + 类比 + 关联课程，随内容扩充（预计 250+ 条）。
- 正文用 `<gd data-term="id">术语</gd>` 标记，点击弹小卡片（定义 + 类比 + 去那一课 + ☆收藏），全局事件委托实现。
- 独立「术语表」页（按模块分组 + 搜索）；收藏进「我的术语本」，支持**翻牌复习**（正面名称背面定义，随机抽 5 个，纯回顾不判分）。

### ADR-9 · 图解优先（本站灵魂之二）
电网知识天然是空间/流程型的，关键概念必配**内联 SVG 图**：发输变配用全景、电压等级阶梯、调度五级金字塔、电价构成堆叠、鸭子曲线、源网荷储、电量交易闭环、cap-and-trade 等。SVG 用 CSS 变量取色（自动适配深浅主题），约定 `<figure class="fig">…<figcaption>` 结构；图属于内容，引擎不管图。

### ADR-10 · 无判分原则
全站没有任何"对/错"判定：无小测、无错题本、无分数。复习只有翻牌回顾。引擎中不写判分代码，schema 中无 quiz 字段。

## 六、体验层设计原则

1. **路线图优先**（ADR-7）——零基础最怕"从哪开始"。
2. **轻量快乐层**——streak 连续学习天数、模块完成度进度环、读完一课的轻微正反馈；不做重型游戏化。
3. **轻量复习**——术语本翻牌（随机 5 个），介于"不做"和"SRS"之间。
4. **5 分钟颗粒度硬上限**——一课只讲一个点，超了拆课。
5. **关键数字给直觉**——50Hz、±800kV、一度电……每个数字配类比或量级感受，不裸抛。
6. **组织架构类内容讲"职能逻辑"重于背名单**——部门/处室设置随机构改革常变，课程注明时效口径。
7. **深/浅主题、字号调节、全文搜索、移动端自适应**为硬需求。
8. **不用 emoji（用户 2026-06-11 指定）**——全站 UI 与课程内容一律不出现 emoji 表情；需要图标时用**线性图标**（内联 SVG stroke 描边）或简洁字形符号（☆ ✓ × → 等），与工程蓝图风格一致。
