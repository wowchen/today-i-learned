## 一、需求

构建一个**美式英语学习网站**：

- **语言**：用通俗易懂的**中文**讲解。
- **对象**：**零基础小白**，目标是"一看就懂"。
- **范围**：把英语**全领域**知识讲透（不是只讲某一块）。
- **口音**：以**美式英语（American English）** 为准（美音、美式拼写、美式表达）。
- **独立工程**：完全独立，**不复用**此前的美剧深度解析阅读站引擎（`20260608-drama-reviews/`）。

### 2. "一看就懂"的统一知识点模板

每个知识点都用同一套结构讲，降低认知负担：

1. **一句话是什么**（大白话 + 类比，先不抛术语）
2. **什么时候用**（真实场景）
3. **怎么用**（规则 + 可点读的例句）
4. **中国人易踩的坑**（对比中文 / 常见错误）
5. **30 秒小测**（即时自查）

### 3. 技术主张

- **纯静态**：原生 HTML / CSS / JavaScript，**无框架、无构建**，可双击 `file://` 直接运行。
- **杀手锏 — 点词即听美音**：用浏览器自带的 **Web Speech API（SpeechSynthesis）**，选 `en-US` 嗓音，点任意英文词/句立刻发音。免费、离线、零依赖，对美语学习站近乎必备。
- **配套能力**：交互式音标表（48 音点击发音 + 口型要领 + 例词）、前端随堂小测（选择/填空，纯前端判分）、深/浅主题、全文搜索、字号调节、移动端适配、键盘导航。

### 4. 现实策略

讲透全领域是巨大的内容工程，不可能一次写完。建议：

1. 先搭好**引擎骨架**（导航 / 点读 / 主题 / 搜索 / 小测组件）。
2. 同时把**一个模块做到极致当样板**，确立统一知识点模板。
3. 之后**分模块逐步扩充**（每个模块分步写，避免一次性产出过长）。

## 四、架构决策记录（ADR）

> 2026-06-10 grill-me 深聊产出。以下为**已锁定**决策，动手前的最终依据。

### ADR-1 · 不要后端，纯静态到底
- 定位：**自用 / 小范围学习**，不做账号、不做 UGC、不做产品化。
- 内容读多写极少，天生适合静态；且要保留离线 `file://` 能力，与"需要服务器"互斥。
- 与现金流策略一致：零运维、零成本。

### ADR-2 · 托管用 GitHub Pages（public repo）
- 三端（家 macOS / 公司 Windows / 手机）打开同一个 URL，零安装即用，自动响应式。
- 站点内容跨设备同步 = 家里 `git push`，其它端刷新即最新，天然解决。
- 纯静态代码**照样能 `file://` 跑**，等于白送离线备用。
- 代价：仓库 public（内容是英语教学，无隐私，可接受）。

### ADR-3 · 内容即数据 + 模板渲染（引擎/内容分离）
- 每个知识点是一条**结构化数据**（五段模板：是什么 / 何时用 / 怎么用+例句 / 易踩坑 / 30 秒小测），由统一渲染器吐成页面。
- 收益：模板强制一致、搜索索引自动生成、加内容=加数据不碰引擎、小测与知识点绑定。
- `file://` 兼容：内容用 `<script>` 注册进全局 registry（避开 fetch 的 CORS 限制），不打包、不需服务器。
- 第一批：搭好引擎（导航 / 点读 / 主题 / 搜索 / 小测）+ 把"发音"模块灌满当样板。

### ADR-4 · 发音方案
- **句子 / 单词**：浏览器自带 **Web Speech API（SpeechSynthesis）**，优先挑 `en-US` 本地嗓音（`voice.localService`）保证离线。完全免费、无 key、无配额。
- **48 个孤立音标**：⚠️ SpeechSynthesis 念不准孤立音素，故**起步用「例词驱动」**——点 /æ/ 念 `cat`、`apple` 等例词。
- **预留升级口**：日后要"真·纯音"，走 Wikipedia CC 音素音频（真人）或 eSpeak-NG WASM（可念任意 IPA），均开源免费，数据结构提前兼容。
- 零基础教学顺序：**自然拼读（phonics）先于 48 音标**，避免一上来糊一脸 IPA 劝退。

### ADR-5 · 进度同步 = GitHub 当后端（方案 B，自动无感）
- 浏览器存 **fine-grained PAT**，自动读写一个**独立 private repo**（如 `english-progress`）里的 `progress.json`；GitHub REST API 支持 CORS，纯前端直连，认证后 5000 次/小时足够。
- **不用 Gist**：Gist 只能用 classic token 的 `gist` 权限，会"所有 Gist 通杀"，blast radius 太大。
- Token 安全边界：fine-grained PAT **只勾这一个 repo**、权限仅 `Contents: Read/Write`、**设 90 天过期**，可随时吊销；泄露极限影响=被人看/改学习进度。
- **token 持久化在三端 localStorage**（用户决策 A）：公司 Windows 仅本人使用、无监控，已知风险并接受，换取完全无感同步。

### ADR-6 · 冲突合并 = 按条目智能合并 + 软删除
- 写入不做整文件覆盖：先拉远端最新 → **按条目并集合并**（同一条以 `updatedAt` 时间戳新者为准）；偏好类单值"最后写入为准"；遇 409 重新拉、重新并、再写。
- **删除走软删除 + 墓碑**：删除标记 `removed: true` + 时间戳，避免被另一台"复活"；墓碑数据极小，保留无妨。
- 效果：两台机器同时乱改基本变成非事件，生词本不丢。

### ADR-7 · 进度数据模型（`progress.json`）
起步四件套（均为带 id + 时间戳的小数据，便于合并）：
1. **生词本 / 收藏夹** —— 核心，跨设备同步主要为它。
2. **错题本** —— 小测做错自动收集。
3. **阅读完成标记 + 续读位置** —— **手动**点"标记已读"（不做滚动自动判定，避免误记）。
4. **偏好** —— 主题 / 字号 / 首选嗓音。
- **SRS 间隔重复：暂不做**，数据结构预留位置，日后再加。

## 五、体验层设计原则

> 2026-06-10 对照主题"更简单、更快速、快乐学英语"复盘产出。架构(ADR 1-7)不动，在其上加薄薄一层学习体验设计。

1. **路线图优先、图书馆其次**（更简单）——首页主入口是一条线性引导路径，永远显示"继续学习：你在第 X 步"；九大模块目录降级为查阅入口。零基础最怕"我从哪开始"，决策疲劳是劝退头号杀手。引擎层面只是多一个 path 数据结构（知识点 id 有序列表）。
2. **轻量快乐层**（快乐，原设计零覆盖）——三个小件：连续学习 streak（由 progress.json 时间戳推导）、模块完成度进度环、小测全对微庆祝。不做重型游戏化（积分/徽章商城），自用站不过度设计。
3. **轻量复习**（更快速，补 SRS 砍掉的洞）——"快速复习"入口：随机抽 5 个收藏词/错题翻牌过一遍。纯前端洗牌，介于"不做"和"完整 SRS"之间的甜点位；日后升级 SRS 数据结构不变。
4. **PWA 壳**（手机体验）——`manifest.json` + service worker 缓存，手机"添加到主屏幕"当 App 用、可离线。渐进增强（`file://` 下 SW 不生效但不影响使用），零成本。
5. **知识点颗粒度上限**（更快速，内容纪律）——每个知识点 5 分钟内学完，超了拆成系列。写内容前定下，比写了 50 个后返工便宜。

4. **网络/短信缩略语（lol / btw / asap）** → 文化与语用。
5. **词典与工具怎么用 + 跟读/影子练习法** → 学习导览。"会学"本身是零基础最缺的能力。
