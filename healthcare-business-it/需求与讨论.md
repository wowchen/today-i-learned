# 需求与讨论 · 医疗业务与信息化学习站

记录本站从立项到落地的关键决策,便于回溯。

## 起点

用户:"新建站 医疗业务与信息化"。定位为「今日得到」合集第 15 站,讲医疗业务、行业运营、医保支付与医疗信息化通识。

## 已确认决策(用户拍板)

1. **流程**:先出 3 个视觉 demo,用户选定后再全量建站。
2. **内容侧重**:业务流程、行业运营、信息化系统三条线都要,不能偏科。
3. **规模**:首版全量,约 13 模块 / 100 课。
4. **视觉**:三套 demo 为 A 临床病历 / B 医院运营驾驶舱 / C 互联互通蓝图。用户选择 **"1 的配色 + 3 的设计"**,即 A 的清爽医疗青绿/病历配色 + C 的系统蓝图/互联互通结构。
5. **首批互动工具(3 个)**:
   - 就诊流程模拟器
   - DRG/DIP 教学示意器
   - 医疗缩略语速配
6. **医疗边界**:仅供学习参考,不构成医疗建议、诊断、治疗、用药、医保申报或合规法律意见;不用真实患者信息。

## 落地结果

- 命名空间 `HIT`;localStorage `hit.progress.v1`;主色 `#0f766e`,辅色 `#0f5c8a`。
- 13 模块 / 100 课 / 76 术语 / 16 角色图鉴 / 16 时间节点 / 缩略语速查 + 流程对照。
- 保留文档式三栏引擎和移动端抽屉;首页增加互联互通蓝图示意。
- 内容由 `tools/`(gen.js + lessons.js + emit.js)单一数据源生成,带校验,便于持续增补。

## 刻意的取舍 / 注意

- **去掉小测**:课程只保留 一句话/讲透/别踩坑/接着读 四段。
- **不用真实病例**:所有实验室和正文例子都是虚构教学示意。
- **DRG/DIP 与政策只讲入门逻辑**:不声称等同真实分组器或本地医保规则。
- **角色图鉴不做个人权威背书**:以医疗业务角色、部门、监管与生态角色为主。
