# 能源北斗与时空服务 · 需求与讨论(本站宪法)

记录本站的目标、范围与关键决策(ADR),后续开发不要违背。

## 目标

用大白话讲透「北斗时空服务(PNT)+ 能源/电力」的通识,打通"卫星导航圈"和"电力圈"两套行话,零基础读者一站学会"能源北斗"是怎么回事。

## 范围

- **讲**:北斗原理(星座/信号/定位/授时)、时空基准、高精度增强(RTK/地基增强/PPP/PPP-B2b)、以及电力五大应用域(授时同步、巡检运维、形变监测、资产作业、短报文应急)、时空数据与数字电网、安全自主与未来。
- **不讲**:继电保护整定、潮流计算、通信协议报文、设备选型、卫星工程内部细节、荐股。

## 关键决策(ADR)

- **ADR-1 引擎复用**:克隆同族 `power-grid-fundamentals` 的成熟数据驱动引擎,命名空间 `PGF→EBD`,localStorage `pgf.*→ebd.*`。不改引擎逻辑,只换内容与视觉。
- **ADR-2 五段式、无小测**:课程用 `what/why/how/pitfalls/links`,沿用同族"无判分"约定。
- **ADR-3 内容即数据**:课程是 `EBD.registerLesson(...)` 的 JS 对象,`<script>` 加载、零构建、`file://` 可用。
- **ADR-4 术语点查**:行业术语首次出现用 `<gd data-term>`,引擎自动弹卡 + 收藏。
- **ADR-5 图解优先**:流程/结构/曲线配内联 SVG,只用预设取色类,自动适配深浅主题,不写死颜色。
- **ADR-6 视觉风格**:先出 3 个 demo(A 星空司南 / B 时空蓝图 HUD / C 国测红金),**定稿 A 星空司南**(深空靛蓝 + 北斗七星金 + 衬线标题 + 星点底纹),选型稿留在 `design/`。
- **ADR-7 规模**:12 模块 / 100 课,comprehensive 全量。
- **ADR-8 中国语境 + 时效**:制度数字以中国为准,会过时的写量级 + 年份口径。
- **ADR-9 安全与隐私语气**:人员定位强调合规边界;电力设施位置属敏感信息,审慎表述。

## 体验原则

大白话先行、一课一个点(≤5 分钟)、关键数字给锚点、误解段写"真误解"、无 emoji、纯静态离线、深浅主题可切换。
